Objectives and Rules

  • No Frameworks.
    • ...That impose their view of the world on us.
    • ...With funny quirks that have to be worked around.
    • ...That require specialized training.
    • ...That become out-dated and hard to remove (lock in).
    • ...That are difficult to refactor in a polystylistic environment.
  • Every component is a DOM object.
  • Every component has an API similar to raw HTML object such as <div>, <form>, and <video>.
  • Every component generates events on self that bubble up the DOM tree.
  • Every component responds to events on self that bubble up from children or are activated directly on it by parents.
  • Every component should be easy to test drive.
  • No component may reach out to the DOM except to TRIGGER events on children.
  • All DOM objects are constructed synchronously - all non-dependent asynchronous behavior is parallel.
  • Composition within the DOM is done using plain HTML by observing node creation.
  • Composition of behavior is done using plan Javascript by observing events bubbling up the DOM tree.
